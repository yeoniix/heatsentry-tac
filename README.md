# heatsentry-tac
군용 집단훈련 온열 질환 예방 및 강제 대응 관제 시스템 

HeatSentry는 KCTC, 전술행군, 야외 전술 훈련 등 고강도 군 훈련 상황에서 병사의 생체 신호와 활동 패턴을 실시간으로 분석해 전투 지속능력 저하, 탈진, 열스트레스, 낙상, 무반응, 낙오 위험을 조기에 탐지하는 AI기반 전술 훈련 안전 관제 시스템입니다. 

단순히 심박수나 체온을 표시하는 웨어러블 알림 장치가 아닌 전술 중 병사의 위치와 생체 상태를 분석하고, 위험 상황이 감지되었을 때 지휘관의 대시보드와 단계적 경고 체계를 통해 조치를 유도하는 분대 단위 전술 안전 관제 시스템이다.

## 🙄 문제정의

전술 훈련에서는 병사가 뛰고, 걷고, 포복하고, 은폐하고, 장시간 대기하는 등 다양한 전술 활동을 수행합니다. 따라서 단순히 심박수가 높다는 이유만으로 위험하다고 판단할 수 없습니다. 중요한 것은 현재 활동 강도에 비해 생체 반응이 비정상적인지를 판단하는 것입니다. 

HeatSentry는 다음과 같은 상황을 조기에 탐지하는 것을 목표로 합니다.
- 활동량은 낮아졌지만 심박수가 계속 높은 경우
- 고강도 기동 이후 심박 회복이 지연되는 경우
- 피부 온도가 지속적으로 상승하는 경우
- 충격 감지 이후 움직임이 없는 경우
- 분대 이동 중 특정 인원이 뒤쳐지거나 경로를 이탈하는 경우
- 위험 상태 직후 통신이 끊긴 경우
- 마지막 위치 기준으로 낙오 또는 고립 가능성이 있는 경우

## ⚙️ 주요 기능

1. 웨어러블 센서 데이터 수집


   HeatSentry는 병사가 착용한 웨어러블 센서 노드로부터 실시간 데이터를 수집합니다.
   수집 대상 데이터는 다음과 같습니다.
   - 심박수
   - 피부온도
   - IMU 가속도
   - IMU 자이로
   - GPS 위도/경도
   - GPS 정확도
   - 배터리 상태
   - 통신 신호 상태
   - 마지막 수신 시각
   
   웨어러블 디바이스는 전술복 어꺠 스트랩에 부착할 수 있도록 설계합니다.

2. 활동 강도 분류


   IMU 시계열 데이터를 기반으로 병사의 현재 활동 상태를 분류합니다.
   분류 대상은 다음과 같습니다.
   - Rest: 정지 또는 대기
   - Low Activity: 저강도 이동
   - High Activiry: 고강도 이동
   - Impact: 충격 또는 낙상 의심
   - No Resonse: 무반응 상태
  
3. 활동 - 생체 불일치 탐지


   HeatSentry의 핵심 AI로직은 활동 강도와 생체 신호의 관계를 분석하는 것입니다.
   전술 훈련 중에는 심박수가 높다는 사실만으로 위험 판단이 어렵기에 다음 상황을 분석합니다.
   - 고강도 활동 + 높은 심박 -> 정상적 전술기동
   - 저활동 상태 + 높은 심박 + 피부온 상승 -> 회복 지연 또는 열 스트레스 위험
   - 충격 감지 + 이후 움직임 없음 -> 낙상 또는 부상 위험
   - 위험 상태 + 위치 이탈 + 통신 두절 -> 낙오 또는 고립 위험
  
4. 전투지속능력 점수 산출


   HeatSentry는 각 병사의 상태를 0~100점의 전투지속능력 점수로 표시함니다.
   점수 산출에는 다음 요소를 활용합니다.
   - 개인 기준 심박 대비 상승률
   - 피부 온도 변화 추세
   - 활동 강도
   - 심박 회복 지연
   - 무반응 지속시간
   - 충격 감지 여부
   - 분대 중심으로부터의 이탈 거리
   - 통신 미수신 시간
   - 마지막 위치 및 마지막 위험도
  
   점수의 기준은 다음과 같습니다.
   - 80~100: 정상
   - 60~79: 주의
   - 40~59: 경고
   - 0~39: 위험

   예시:
   A03 병사
   전투 지속능력: 28/100
   상태: 위험

   위험 원인:
   - 심박 과상승
   - 고강도 기동후 활동량 급감
   - 피부온도 지속상승
   - 분대중심으로부터 92m 이탈
   - 마지막 수신 58초 전

5. 실시간 태블릿 대시보드


   대시보드에선 각 병사의 위치와 상태를 지도 위에 색상과 함께 실시간으로 표시합니다.
   정상: 초록색
   주의: 노란색
   경고: 주황색
   위험: 빨간색
   통신두절: 회색

   주요 구성 화면은 다음과 같습니다.
   - 지도 기반 병사 위치 관제 화면
   - 분대 전체 상태 화면
   - 병사별 상세 상태 화면
   - 전투 지속능력 점수 변화 그래프
   - 심박 및 피부 온도 변화 그래프
   - 활동 강도 변화 그래프
   - 위험 원인 설명 패널
   - 경고 확인 및 조치 입력 화면
   - 훈련 후 AAR로그 화면

     대시보드는 태블릿 브라우저에서 접속할 수 있도록 설계하며 실시간 데이터 갱신을 위해 WebSocket통신을 활용합니다.

6. 단계적 경고 에스컬레이션


   위험 상태가 감지되면 시스템은 단계적으로 경고를 전파합니다.
   - 1단계: 착용자 진동 경고
   - 2단계: 분대장 또는 현장 통제자의 대시보드 경고
   - 3단계: 조치 내용 경고 이력 로그 저장
   - 이를 통해 훈련 중 위험 신호가 단순히 표시되는 것에 그치지 않고 실제 조치로 이어질 수 있도록 설계합니다.


7. AAR안전 로그 분석


   훈련 종료 후 AAR, 즉 사후강평과정에서 안전 관련 의사결정과 조치 지연 여부를 분석가능합니다.
   AAR에선 다음과 같은 정보를 제공합니다.
   - 훈련 중 위험 이벤트 타임리인
   - 병사별 전투 지속능력 변화
   - 위험 발생 위치
   - 조치까지 걸린 시간
   - 통신 두절 및 낙오 의심 구간
   - 분대 이동 경로와 이탈 지점
  

## 🤖 AI 파이프라인
HeatSentry는 안정성과 확장성을 위해 하이브리드 AI구조를 사용합니다.

1. Rule-Based Risk Engine


   초기 안전 판단과 시연 안정성을 위한 규칙 기반 위험도 판단 엔진입니다.
   예시 규칙은 다음과 같습니다.
   - 충격 감지 후 일정 시간 움직임이 없으면 낙상 위험
   - 활동량이 낮은데 심박수가 높으면 회복 지연 위험
   - 피부 온도가 지속적으로 상승하면 열 스트레스 위험
   - 분대 중심으로부터 일정 거리 이상 이탈하면 낙오 위험
   - 통신 끊김 직전 위험도가 높았다면 고립 위험
  
2. ML Model


   IMU, 생체 신호, GPS데이터에서 추출한 Feature를 기반으로 활동 상태와 위험 상태를 분류합니다.
   사용 가능 모델은 다음과 같습니다.
   - Random Forest
   - XGBoost
   - Logistic Regression
   - SVM
  
   활용 Feature 예시는 다음과 같습니다.
   - 가속도 RMS
   - 가속도 표준편차
   - Jerk
   - Peak Frequency
   - 심박 Z-score
   - 피부온도 상승 기울기
   - 무반응 지속시간
   - 이동속도
   - 분대 중심으로부터의 거리
   - 통신 미수신 시간
  
3. Deep Learning Time-Series Model


   IMU 원시 시계열 데이터를 입력으로 받아 활동 상태를 분류하는 딥러닝 모델입니다.
   후보 모델은 다음과 같습니다.
   - 1D CNN
   - CNN-GRU
   - CNN-LSTM
   초기에는 Random Forest기반 모델을 기준으로 구현하고 이후 1D CNN기반 시계열 분류 모델로 고도화합니다.

4. Activity-Biometric-Location Mistmatch Detection


   HeatSentry는 활동강도, 생체신호, 위치 데이터 3가지의 데이터의 조합을 통해 위험상태를 판단합니다.
   - 활동 강도 감소 + 심박 회복 지연 + 피부 온도 상승 = 탈진 or 열스트레스 위험
   - 위험도 상승 + 분대 중심 이탈 + 통신 미수신 = 낙오 또는 고립 위험
   - 충격 감지 + GPS위치 정지 + IMU무반응 = 낙상 또는 부상 위험
